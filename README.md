# 統合投資ダッシュボード (Investment Dashboard)

日本株・ETF・オプション等の市場データを自動収集・集計し、フロントエンド側で動的に分析チャートを描画する一元管理ダッシュボードです。

---

## 📊 ダッシュボード・ページの仕様

### 1. 空売り集計ダッシュボード (`short_selling.html`) [最新アップデート]
東証の空売り集計データと日々の売買代金をもとに、市場の底打ちやセンチメント転換を分析するページです。

#### 📈 UI・チャートレイアウト仕様
*   **2カラムレスポンシブ・レイアウト**:
    *   PC表示帯では「売買フロー推移」と「空売り内訳推移」が左右2カラムで並んで表示される（`grid-cols-1 lg:grid-cols-2`）。
    *   スマホなどのモバイル表示では縦並びとなる。
*   **チャートの高さと余白の厳密な統一**:
    *   両方のチャート高さを `420px` に統一。
    *   X軸・Y軸が上下・左右で綺麗に整列するよう、`margin={{top:15, right:25, left:-15, bottom:0}}` 等の設定を完全に同期させている。
    *   バーチャートの幅感も統一（Breakdownチャート側の `barCategoryGap` を削除して幅を合わせる）。
*   **デフォルトの表示期間 (Brush)**:
    *   直近の動きがすぐ確認できるよう、初期状態では「直近20日」にズームされている。

#### 🎨 独自のデザイン・装飾ルール
*   **売買代金「最大値」の強調 (Flow Chart)**:
    *   売買代金のバー（`total`）は原則として「薄紫 (`#a78bfa`, opacity:0.5)」で描画されるが、**過去最大売買代金を記録した日（`mx_t_d`）のバーのみ「濃い紫 (`#7C4DFF`, opacity: 1.0)」に変化**し、視覚的な基準日として目立たせる構成。
    *   ※ 現物売り（`genbutsu_uri`）のバーは原則として濃灰色となるように逆転させている。
*   **ピークドット (Breakdown Chart)**:
    *   空売り比率が「直近20日以内で最も高かった日」に対して、ピンク色の `Peak!` ドット（`#EC4899`）と文字を付与する。
    *   ※ 「売買フロー推移」側のチャートにはチャートをスッキリさせるためドットアイコンは表示しないルールとなっている。
*   **モジュール化されたヘルプUI**:
    *   長大なテキストを並べるのではなく、各カードタイトルの横にある `?` アイコンをクリックした際に対応する Modal（ポップアップ）を開き、ヘルプやシグナルの仕様を読むことのできるクリーンな設計を採用している。

#### 🚦 シグナル判定ロジック
以下の2つの先行指標を用いて、直近20日間のデータから『市場の底打ち（買い転換）』を判定している。
1.  **売買代金膨張シグナル (S1)**: 当日の売買代金が、直近20日移動平均に対して「1.3倍（+30%）」以上乖離しているか。（パニック売りクライマックスや出来高急増を検知）
2.  **空売り比率ピークアウト (S2)**: 当日の空売り比率が、直近20日内の最大比率（ピーク）から相対的に「20%以上低下」しているか。

### 2. セクター分類分析 (`sector_category.html`)
全セクターを7カテゴリに分類し、左：パフォーマンスチャート / 右：ランキング の2カラム構成で表示。

*   **データ更新**: 市場時間中（前場・後場）において、ブラウザ側で `5分ごと` にJSONデータを非同期再フェッチして自動更新する。
*   **パフォーマンス計算基準**:
    *   イントラデイ（1d〜14d）: 前日終値（baseIndex = startIndex - 1）を0%として算出。
    *   日次（1mo〜1y）: 期間開始日そのものを基点とする。
*   **タイムスタンプ処理**: yfinanceから取得したデータはUTCで統一されているため、フロントエンド（JS）側で `+9h` の処理を行い JST に変換する。この `+9h` を外すと rangebreaks（土日除去など）のロジックが破綻しチャートが消えるため**厳禁**。
*   **チャート参照線の使い分け**:
    *   米国の前日確定結果は「実線（NASDAQ100など）」で表示。
    *   リアルタイムに動く「限月付き先物（NQM, ESM等）」は「破線」で表示し、現在の日中の動きを可視化。

## ⚙️ GitHub Actions と データ更新アーキテクチャ

本プロジェクトでは GitHub Actions を基盤とし、外部サービスに依存せずデータ更新から自動デプロイまでを完結させている。

### ワークフロー一覧と実行タイミング

| ファイル | 名前 | 実行タイミング | デプロイ方式 |
|---|---|---|---|
| `intraday_etf.yml` | イントラデイETF更新 | 市場時間中 **15分ごと連続ループ** | mainにcommit + gh-pages直接push |
| `daily_etf.yml` | 日次ETFデータ取得 | **毎日 JST 16:00** | peaceiris（keep_files: true） |
| `daily_participant.yml` | 手口・オプション取得 | **毎日 JST 20:05** | peaceiris（keep_files: true） |
| `daily_short_selling.yml` | 空売り集計更新 | **毎日 JST 16:30** | peaceiris（keep_files: true） |
| `daily_task.yml` | その他タスク・フルデプロイ | 毎週木曜 JST 18:00 + HTML/yml変更時 | peaceiris（keep_files: true） |

### 🔥【最重要】 GitHub cron不発問題と「ループ型アーキテクチャ」の解答
GitHub Actionsの `schedule` (cron) は高負荷時に発火率が極端に落ちる（過去実績で約21%しか起動しなかった）という致命的問題を抱えていた。市場のリアルタイムデータを扱う上でこれに対処するため、以下のアプローチを開発した。

*   **解決策： セッション起動型ループ（intraday_etf.yml）**
    *   cron を「前場開始（JST 9:00付近）」および「後場開始（JST 12:30付近）」の着火材として複数仕掛ける。
    *   一度 Actions が立ち上がると、Python スクリプト終了後、yml 内で `sleep 900`（15分）を挟み、市場が終わる時刻になるまで**自己完結で無限ループ（再実行）を行う**。
    *   これにより、GitHub側のcron不発問題に依存せず、確実に15分ごとのデータ取得→コミット→Pages配信が担保されている。

### ⚠️ 過去のCDN・デプロイに関する教訓
*   **CDNは404を強力にキャッシュする**:
    *   gh-pages にファイルが存在しても、一度ブラウザ・CDNが 404 を返してしまうと数時間復旧しない。ファイル新規作成時は要注意。
*   **main ブランチへのコミットが原則**:
    *   当初一時ファイル群を `gh-pages` への直pushで解決しようとしたがパイプラインが乱れた。原則データ（JSON等）はすべて `main` に commit し、それをもとに Pages がビルドされる構成が最も安定する。

## 🎨 デザイン仕様 (Design Reference)

### カラースキーム（全体）

| 用途 | カラーコード | Tailwind相当 |
|------|------------|----------|
| **背景 (Body)** | `#f8fafc` | slate-50 |
| **カード背景** | `#ffffff` | white |
| **境界線/グリッド** | `#e2e8f0` | slate-200 / slate-100 |
| **メインテキスト** | `#334155` | slate-700 |
| **注釈・軸数値** | `#94a3b8` | slate-400 |
| **Primary (紫)** | `#7C4DFF` | カスタム紫 (Accent) |
| **ポジティブ/買い** | `#10B981` | emerald-500 |
| **ネガティブ/売り** | `#F43F5E` | rose-500 |

### `short_selling.html` 専用 厳格カラー定義 (`COLORS`オブジェクト)
分析を視覚的にわかりやすくするため、チャートの色使いは以下の定義を死守する。

*   **最大売買代金 (total 強調)**: `#7C4DFF` （不透明度 1.0 にして極端に目立たせる）
*   **通常日の売買代金 (sell)**: `#a78bfa` （不透明度 0.5 の薄紫で抑える）
*   **現物売り (totalBar)**: `#94a3b8` （灰色にして総売り・差分を表現）
*   **空売り合計 / 機関空売り (short / kikan)**: `#F43F5E` (rose-500)
*   **規制なし空売り (free)**: `#fda4af` (rose-300 色調整)
*   **空売り比率 (ratio)**: `#475569` (slate-600 のダークグレー・折れ線)
*   **Peakドット (peakDot)**: `#EC4899` (pink-500 の強烈なピンク)

    ├── daily_participant.yml           # [デイリー] 手口・オプション取得
