# 02. データ分析スキル (Analysis)

`01-data-skill.md` にて取得された最新のJSONデータを読み込み、ユーザーの指定した「分析期間」に合わせて、以下のロジックに従って相場の解釈（定性・定量分析）を行ってください。

## 【最重要】分析期間に応じたデータの動的スライスとソース自動選択
ユーザーから「今日の分析」「1週間の分析」「3週間の分析」などの指示があった場合、以下のルールに従って起点（比較元の価格）を決定し、パフォーマンス（％）を計算してください。

1. **期間による比較対象日（起点）の自動計算ルール**
   * **今日（Daily）**: 本日の始値（または前日終値）と直近最新値を比較。
   * **1週間（Weekly）**: 直近5営業日前の終値を起点とし、最新値と比較。
   * **3週間（3 Weeks）**: 直近15営業日前の終値を起点とし、最新値と比較。
   * （その他の「N週間やNヶ月」指定時も同様に、N×5営業日前、あるいはN×20営業日前のデータを起点として計算してください）

2. **参照するJSONデータの自動スイッチングルール**
   * **2週間（10営業日）以内の期間指定の場合**: `fetch_intraday.py` が生成した `data/etf_intraday.json`（最大14日間の5分足データ）を用い、解像度の高いリアルタイムな騰落率を計算する。
   * **3週間（15営業日）以上の期間指定の場合**: Yahoo APIの都合上イントラデイデータが足りないため、必ず `etf_data_manager.py` が生成した `data/etf_data.json`（過去400日分の日足データ）に切り替え、当該日数分遡った終値を起点としてパフォーマンスを計算する。

## 分析ロジック要件
1. **売買フロー底打ち検知 (`short_selling.json`)**:
   売買代金の膨張シグナル（S1：20日MA比+30%超）、および空売り比率ピークアウトシグナル（S2：直近ピークから相対で20%低下）を計算し、「Phase（下落 / 反発準備 / 反転上昇）」の状況を判定する。（期間指定に関わらず常に最新の20日MAやPeakを用いて直近の状況を判定）
2. **米系投資銀行のオプション手口分析 (`teguchi.json`, `option_history.json`)**:
   ゴールドマンサックス、J.P.モルガン、モルガン・スタンレーなど「米系投資銀行」のオプション・先物手口に絞って建玉の偏りを解析し、彼らが想定している相場の上値抵抗線（レジスタンス）と下値支持線（サポート）、および仕掛けの意図を推測する。
3. **セクター分析 (`etf_data.json` 等)**:
   必ず `etf_data_manager.py` が生成した日次データを優先的に用い（または期間に応じて `etf_intraday.json` を用い）、主要セクターETFの指定期間における絶対値パフォーマンスを比較し、「現在日本市場で最も買われているセクター」と「最も売られている足枷のセクター」を明確に判定する。
4. **個別株ランキング (`etf_intraday.json` または `etf_data.json`)**:
   要求された指定期間に応じたパフォーマンス差分を計算し、TOPIX100全銘柄の中から **「ランキング上位20銘柄」** と **「下位20銘柄」** を正確に抽出する。
5. **各種インデックスの評価 (`etf_intraday.json` 等)**:
   US先物指数（NQmain, ESmain, YMmain等）および、日本主要指数（日経平均 ^N225, TOPIX 1306.T, 半導体ETF 2644.T）の指定期間における騰落結果（%）を取得・評価する。
   
**終了条件**:
上記の解析から「指定期間における相場のファクトと考察」が全て揃った段階で、次のレポート出力ステップ（03）へ進んでください。
